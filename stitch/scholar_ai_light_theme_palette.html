# Design System: Scholar AI (Light Theme) The Light Theme for LLMBase, titled **"The Scholarly Synthesis,"** is designed to feel like a high-end digital library or an academic journal. It prioritizes clarity, whitespace, and a focused reading environment. ## 1. Core Color Palette ### Backgrounds & Surfaces - **App Background:** `#F8FAFC` (Slate 50) - A clean, off-white base to reduce eye strain. - **Surface Layer 1 (Sidebar):** `#F1F5F9` (Slate 100) - Subtle separation for the navigation area. - **Surface Layer 2 (Cards/Containers):** `#FFFFFF` (White) - Pure white for primary content areas to make text pop. - **Surface Layer 3 (Modals/Overlays):** `#FFFFFF` with a soft `#0F172A/05` shadow. ### Typography - **Primary Text (Headings/Body):** `#0F172A` (Slate 900) - Near-black for maximum readability. - **Secondary Text (Summaries/Metadata):** `#475569` (Slate 600) - Muted for visual hierarchy. - **Muted Text (Captions/Placeholders):** `#94A3B8` (Slate 400) - De-emphasized labels. ### Accents & Actions - **Primary Brand (Indigo):** `#4F46E5` (Indigo 600) - Used for primary buttons, active nav states, and wiki-links. - **Secondary Accent (Cyan):** `#0891B2` (Cyan 600) - Dedicated to AI-driven features like Q&A and Citations. - **Success (Emerald):** `#059669` (Emerald 600) - For healthy status indicators. - **Warning (Amber):** `#D97706` (Amber 600) - For linting issues or missing metadata. - **Error (Rose):** `#E11D48` (Rose 600) - For broken links or ingestion failures. ## 2. Typography System - **UI Font:** `Inter` - Variable font for precise control over weights. - **Body Text (CJK):** `Noto Serif SC` - Provides an elegant, literary feel for long-form reading. - **Code Snippets:** `JetBrains Mono` - Clear, monospaced font for technical content. ## 3. UI Components & Effects - **Border Radius:** 12px for cards, 8px for buttons and inputs. - **Borders:** `#E2E8F0` (Slate 200) for subtle dividers and card outlines. - **Shadows:** Soft, diffused shadows (e.g., `shadow-sm`, `shadow-md`) to create depth without clutter. - **Transitions:** 150ms ease-in-out for hover states and transitions.
